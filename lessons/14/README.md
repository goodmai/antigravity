# Урок 14: Микросервисы и Docker

Разработка распределенных систем требует оркестрации. Antigravity берет на себя рутину управления контейнерами.

---

## 🐳 Docker Management

Агент понимает контекст вашего приложения (Node.js, Python, Go) и может:

1.  **Генерировать Dockerfile**: Оптимизирует слои для кэширования.
2.  **Создавать docker-compose.yml**: Настраивает сети, тома, переменные окружения.
3.  **Оркестрация**: Команда _"Подними мне локальное окружение"_ приведет к запуску `docker-compose up` и проверке здоровья сервисов.

---

## 🔧 Диагностика

Если сервис А не видит сервис Б:
Агент прочитает логи обоих контейнеров, проверит настройки сети в compoise-файле и скажет: _"У тебя закрыт порт 8080 в контейнере API, давай я открою его"_.

---

## 📝 Задание

1.  Создайте простой файл `server.js` (Express).
2.  Попросите агента: _"Оберни это приложение в Docker и создай docker-compose файл для его запуска"_.
3.  Попросите агента запустить контейнер и проверить, что он отвечает.
