Урок 1: Режимы агента Antigravity

В этом уроке мы разберем основные режимы работы агента Antigravity и способы переключения между ними.

Режимы диалога (Conversation-Level Modes)

При начале нового диалога вы можете выбрать один из режимов:

  1. Planning (Планирование)
    • Агент планирует действия перед выполнением.
    • Используется для глубокого исследования, сложных задач или совместной работы.
    • В этом режиме агент организует работу в группы задач, создает артефакты и тщательно продумывает план для достижения наилучшего качества.
  2. Fast (Быстрый)
    • Агент выполняет задачи напрямую.
    • Используется для простых задач, которые можно выполнить быстро (например, переименование переменных, запуск пары команд bash).
    • Полезно, когда важна скорость, а задача достаточно проста, чтобы не беспокоиться о снижении качества.

Указания для ИИ (Rules for AI)

Вы можете настраивать поведение агента (включая язык общения) через Глобальные правила:

Глобальные правила (Global Rules) * Согласно документации, главный системный скрипт расположен в файле ~/.gemini/GEMINI.md. * Эти правила применяются ко всем вашим рабочим пространствам (workspaces). * Пример: “Всегда отвечай на русском языке”, “Соблюдай стиль Google Style Guide”.

Агент автоматически учитывает эти правила при работе в любом проекте.

Правила рабочего пространства (Workspace Rules) * Располагаются в папке .agent/rules в корне вашего воркспейса или git-репозитория. * Для каждого правила можно настроить способ его активации: * Manual: Активируется вручную через упоминание @ в строке ввода. * Always On: Правило применяется всегда. * Model Decision: Модель сама решает, применять ли правило, исходя из его текстового описания. * Glob: Правило применяется к файлам, подходящим под маску (например, *.js или src/**/*.ts).

Агент комбинирует глобальные и локальные правила для достижения наилучшего результата.

Настройки агента (Settings)

Настройки, общие для всех диалогов, находятся на вкладке “Agent” в панели настроек.

Artifact Review Policy (Политика проверки артефактов)

Опции политики проверки:

  • Always Proceed (Всегда продолжать): Агент никогда не запрашивает проверку.
  • Request Review (Запрашивать проверку): Агент всегда спрашивает разрешение.
    • Если агент решает запросить проверку плана реализации, он остановится и уведомит вас, позволяя изучить план и добавить комментарии.

Terminal Command Auto Execution (Авто-выполнение команд)

Для инструмента генерации терминальных команд:

  • Request Review: Никогда не выполнять команды автоматически (кроме тех, что в списке разрешенных).
  • Always Proceed: Никогда не спрашивать (кроме тех, что в списке запрещенных).

Списки разрешенных (Allow list) и запрещенных (Deny list) настраиваются в той же вкладке “Agent”. Примечание: изменения настроек применяются только к новым сообщениям.

Agent Non-Workspace File Access (Доступ к внешним файлам)

Разрешение агенту просматривать и редактировать файлы вне текущего рабочего пространства.

  • По умолчанию агент имеет доступ только к файлам в воркспейсе и корневой папке приложения ~/.antigravity/ (где хранятся артефакты и память).
  • Используйте с осторожностью, чтобы не раскрыть секретные данные.

Пример настройки: Если вы хотите разрешить агенту доступ к директории с проектами Node.js, вы можете добавить путь ~/node/ в список разрешенных путей.

Внутренние режимы Agentic Mode

Внутри агентного режима работа делится на три фазы:

  • PLANNING (Планирование):
    • Анализ требования.
    • Создание implementation_plan.md.
    • Согласование плана с пользователем.
  • EXECUTION (Выполнение):
    • Написание кода.
    • Создание файлов.
    • Выполнение команд терминала.
  • VERIFICATION (Проверка):
    • Запуск тестов.
    • Проверка работоспособности.
    • Создание walkthrough.md (отчета о проделанной работе).

Как переключать режимы?

Агент самостоятельно управляет переключением внутренних режимов, но вы можете направлять его:

  • Чтобы начать планирование: Попросите составить план действий (“Make a plan for…”).
  • Чтобы перейти к выполнению: Дайте команду на реализацию (“Go ahead”, “Execute the plan”).
  • Чтобы запустить проверку: Попросите протестировать результат (“Verify the changes”).

Практическое задание

  1. Попросите агента создать план для написания простого “Hello World” на Python. Обратите внимание, как агент перейдет в режим PLANNING.
  2. Утвердите план. Агент перейдет в EXECUTION.
  3. Попросите проверить работу скрипта. Агент перейдет в VERIFICATION.
  4. Проверка приоритета правил: Создайте в текущем воркспейсе правило (Workspace Rule) в папке .agent/rules/, которое велит агенту отвечать на испанском. Затем поздоровайтесь с агентом. На каком языке он ответит, если в Глобальных правилах (Global Rules) у вас стоит русский? Это поможет понять, какой уровень правил имеет более высокий приоритет.

Отключение правил

Для выключения локального правила в Antigravity без потери наработанного контента, откройте соответствующий файл в .agent/rules и обновите секцию метаданных:

```markdown

trigger: manual

##Запрет на удаление## Шаг 1: В правом нижнем углу есть Antigravity-Settings-Advanced Settings. В окне Agent найдите Deny List Terminal Commands

Шаг 2: Проверьте синтаксис Deny List. Убедитесь, что после ввода rm в поле Deny List вы нажали Enter или кнопку + Add, чтобы значение сохранилось (на скриншоте оно выглядит как введенный текст в поле, но не как добавленный токен).

Шаг 3: Смена режима (Рекомендуется). Для критических операций лучше сменить Terminal Command Auto Execution - Always Proceed на Request Review. В этом случае агент будет спрашивать разрешение на любую команду, что исключает случайные удаления.
