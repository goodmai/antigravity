Урок 18: Рефакторинг и Работа с Legacy Кодом

Antigravity открывает новые горизонты в рефакторинге благодаря огромному контекстному окну моделей Gemini Pro (до 2 млн токенов). Это позволяет загружать в память модели целые модули или даже небольшие проекты целиком для глубокого анализа и переработки.

📚 Что такое AI-Refactoring?

AI-Refactoring — это процесс изменения внутренней структуры кода без изменения его внешнего поведения, выполняемый автономным агентом под контролем архитектора (человека).

В отличие от классического рефакторинга (“Rename Variable” в IDE), AI-рефакторинг оперирует семантикой:

  • “Перепиши этот класс в функциональном стиле”
  • “Выдели бизнес-логику из контроллера в сервисный слой”
  • “Замени синхронные вызовы на асинхронные”

Best Practices (Лучшие практики)

  1. Iterative Approach (Итеративность): Не просите переписать весь проект сразу. Делите задачу на модули.
  2. Test-Driven Refactoring (TDD): Перед рефакторингом убедитесь, что у вас есть тесты. Лучшая стратегия:
    • Попросить агента написать тесты для старого кода (Snapshot Testing).
    • Выполнить рефакторинг.
    • Убедиться, что тесты проходят.
  3. Context Isolation: Используйте .antigravityignore или просто не открывайте лишние файлы, чтобы не размывать внимание модели.
  4. Role Prompting: Задавайте агенту “роль”. “Ты — Senior Java Developer, эксперт по Spring Boot”.

🐍 Примеры Промптов: Python

1. Переход на Pydantic V2

“Проанализируй файл models.py. Перепиши все модели данных с использованием Pydantic V2. Замени устаревшие методы dict() на model_dump(), а parse_obj на model_validate. Убедись, что все поля имеют тайп-хинты.”

2. Оптимизация pandas

“Этот цикл for по датафрейму очень медленный. Рефактори функцию calculate_metrics, используя векторизированные операции pandas или numpy. Сравни время выполнения старой и новой версии.”

3. Asyncio Migration

“Перепиши этот синхронный Flask routes handler на асинхронный стиль для FastAPI. Замени блокирующие вызовы БД (SQLAlchemy) на await session.execute(...).”

☕ Примеры Промптов: Java

1. Java 8 to Java 17+

“Рефактори этот класс. Замени длинные цепочки проверок if (obj != null) на использование java.util.Optional. Там где возможно, используй var и новые конструкции switch (pattern matching), доступные в Java 17.”

2. Удаление Lombok (De-Lombok)

“Наша политика безопасности запрещает использование Lombok. Сгенерируй явные геттеры, сеттеры, конструкторы и методы equals/hashCode для класса UserDto, сохранив неизменяемость (Immutability) полей.”

3. Тесты на JUnit 5

“Мигрируй этот тестовый класс с JUnit 4 на JUnit 5. Замени @Before на @BeforeEach, Assert.assertEquals на Assertions.assertEquals. Используй assertThrows вместо expected в аннотации @Test.”

🐹 Примеры Промптов: Go

1. Обработка ошибок

“Проведи рефакторинг функции ProcessData. Избавься от глубокой вложенности if err != nil. Используй паттерн ‘early return’ (guard clauses). Оберни ошибки с контекстом, используя fmt.Errorf(\"failed to process step X: %w\", err).”

2. Interface Segregation

“Структура BigService имеет слишком много методов. Проанализируй её использование в пакете handlers и выдели малые интерфейсы (Reader/Writer), следуя принципу Interface Segregation Principle (ISP).”

3. Concurrency Patterns

“Этот код запускает горутины в цикле без контроля. Перепиши его с использованием errgroup.Group для ограничения параллелизма и корректной обработки первой возникшей ошибки.”

📝 Задание

  1. Выберите старый файл в вашем проекте.
  2. Попросите агента: “Проанализируй этот код на предмет ‘code smells’ и предложи план рефакторинга по шагам” (См. Урок 2 про артефакты планов).
  3. Выполните первый шаг плана.
