# Урок 9: Взаимодействие с Browser Subagent

В этом уроке мы разберем один из самых мощных инструментов Antigravity — встроенный браузер и управляющий им **Browser Subagent**.

Когда основному агенту требуется информация из веба или взаимодействие с веб-интерфейсом, он делегирует эту задачу специализированному субагенту.

---

## 🤖 Что такое Browser Subagent?

**Browser Subagent** — это модель, специально обученная работе с веб-страницами. Она отличается от основной модели агента и обладает уникальным набором инструментов для прямой манипуляции браузером.

### Ключевые особенности:

1.  **Специализация**: Субагент понимает структуру DOM, умеет интерпретировать визуальное состояние страницы и логи консоли.
2.  **Автономность**: Он может работать в фоновых вкладках, не прерывая вашу основную работу.
3.  **Безопасность**: Во время работы субагента на активной странице накладывается защитный слой, предотвращающий конфликты между вашими действиями и действиями AI.

---

## 🛠️ Инструментарий субагента

Субагент не просто "смотрит" на страницу, он полноценно управляет ею, используя:

- **Click / Type / Scroll**: Стандартные действия пользователя.
- **DOM Capture**: Чтение структуры страницы для точного поиска элементов.
- **Screenshots & Video**: Визуальное подтверждение действий и анализ UI.
- **Console Logs**: Чтение ошибок и логов разработчика для отладки.
- **Markdown Parsing**: Преобразование сложного HTML в чистый текст для быстрого анализа данных.

---

## 📂 Юзкейсы Browser Subagent

Где помощь субагента незаменима:

### 1. Исследование документации

Если библиотека обновилась и локальные файлы устарели, агент отправит субагента на официальный сайт для получения актуальных API reference.

### 2. E2E Тестирование

Субагент может пройти путь пользователя: залогиниться, добавить товар в корзину и проверить, что кнопка оплаты активна.

### 3. Сбор данных (Data Mining)

Быстрое извлечение цен из каталогов, новостей или спецификаций железа для ваших проектов.

### 4. Автоматизация веб-задач

Например, заполнение сложных государственных форм или миграция данных между двумя SaaS-сервисами, у которых нет API.

---

## ⚠️ Правила взаимодействия

1.  **Синий контур (Blue Border)**: Когда вы видите синюю рамку вокруг окна браузера и панель с описанием действий — это работает субагент.
2.  **Не мешайте**: В этот момент ручное взаимодействие со страницей заблокировано, чтобы не сбить алгоритмы навигации.
3.  **Многозадачность**: Вы можете открыть новую вкладку и продолжать работать — субагент продолжит выполнять задачу в своей вкладке, даже если она не в фокусе.
