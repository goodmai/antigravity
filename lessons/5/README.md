# Урок 5: Типовые Скиллы (Skills)

Скиллы — это модульные пакеты, которые расширяют возможности агента, предоставляя ему специализированные знания, рабочие процессы (workflows) и инструменты. В этом уроке мы разберем основные "типовые" скиллы, которые доступны в вашей среде разработки.

---

## 🎨 Визуальное искусство и дизайн

### 1. `algorithmic-art`

**Описание**: Создание генеративного искусства с помощью p5.js. Использует алгоритмическую философию для генерации уникальных визуальных эффектов.

- **Промпт**: "Создай генеративное искусство на тему 'Северное сияние' с регулятором солнечной активности".
- **Результат**: Интерактивный HTML-файл с холстом p5.js и панелью управления параметрами.
- **Юзеркейсы**: Создание уникальных фонов, визуализация математических концептов, интерактивные арт-объекты.
- **Тесткейс**: Открыть HTML в браузере, убедиться, что анимация плавная и слайдеры меняют изображение в реальном времени.

### 2. `canvas-design`

**Описание**: Создание статического визуального искусства (постеры, обложки) в форматах .png и .pdf.

- **Промпт**: "Создай минималистичный постер в стиле 'Brutalist Joy' для конференции по ИИ".
- **Результат**: Файл .png или .pdf с высоким качеством графики и выверенной типографикой.
- **Юзеркейсы**: Дизайн мерча, афиши, презентационные слайды, обложки документов.
- **Тесткейс**: Проверить отсутствие наложений текста, соответствие цветовой палитре и высокое разрешение экспорта.

### 3. `brand-guidelines`

**Описание**: Применение официальных стилей Anthropic (цвета, шрифты) к любым артефактам.

- **Промпт**: "Стилизуй эту презентацию под брендбук Anthropic".
- **Результат**: Документ с использованием шрифтов Poppins (заголовки) и Lora (текст), а также фирменных цветов (#141413, #d97757 и др.).
- **Юзеркейсы**: Создание корпоративных отчетов, презентаций и внутренних документов в едином стиле.
- **Тесткейс**: Убедиться, что цвета элементов и шрифты соответствуют значениям из `SKILL.md`.

## 📄 Документы и Офис

### 4. `docx` / `pptx` / `xlsx`

**Описание**: Полноценная работа с профессиональными документами: создание, редактирование, анализ, работа с исправлениями (tracked changes).

- **Промпт**: "Создай коммерческое предложение в .docx на основе этого текста" или "Обнови данные в таблице .xlsx".
- **Результат**: Готовый офисный файл. Для `docx` поддерживается режим рецензирования.
- **Юзеркейсы**: Автоматизация документооборота, создание сложных отчетов, финализация юридических соглашений.
- **Тесткейс**: Открыть файл в LibreOffice/Word, проверить структуру, формулы (для xlsx) или стили заголовков.

## 🛠️ Разработка и Интеграции

### 5. `mcp-builder`

**Описание**: Руководство по созданию серверов Model Context Protocol (MCP) для интеграции внешних сервисов.

- **Промпт**: "Собери MCP-сервер для работы с API криптобиржи на TypeScript".
- **Результат**: Структура проекта сервера с определенными инструментами (tools) и схемами ввода-вывода.
- **Юзеркейсы**: Расширение возможностей ИИ-агентов новыми API, создание мостов между локальными данными и ИИ.
- **Тесткейс**: Запустить `npm run build` и проверить сервер через `mcp-inspector`.

### 6. `skill-creator`

**Описание**: Инструмент для создания новых скиллов для Клода. Позволяет упаковывать знания в формат `.skill`.

- **Промпт**: "Создай новый скилл для проведения код-ревью на Python".
- **Результат**: Папка скилла с `SKILL.md`, ресурсами и скриптами.
- **Юзеркейсы**: Создание корпоративной базы знаний, упаковка повторяющихся workflows в удобный формат.
- **Тесткейс**: Запустить `package_skill.py <path>` и убедиться, что валидация прошла успешно и создан `.skill` файл.

## 🌐 Веб-разработка и Тестирование

### 7. `webapp-testing`

**Описание**: Набор инструментов для автоматизированного тестирования веб-приложений с помощью Playwright.

- **Промпт**: "Проверь, работает ли кнопка регистрации на моем локальном сайте".
- **Результат**: Отчет о выполнении теста, скриншоты ошибок, логи консоли.
- **Юзеркейсы**: E2E тестирование, проверка адаптивности, поиск багов в UI.
- **Тесткейс**: Запустить скрипт тестирования и увидеть `Assertion passed` в консоли.

### 8. `web-artifacts-builder`

**Описание**: Создание сложных React-артефактов (с использованием Tailwind, shadcn/ui) и их упаковка в один HTML-файл.

- **Промпт**: "Собери сложный дашборд аналитики на React с графиками и вкладками".
- **Результат**: Самодостаточный `bundle.html`, работающий как интерактивное приложение.
- **Юзеркейсы**: Прототипирование сложных интерфейсов, создание интерактивных калькуляторов и дашбордов.
- **Тесткейс**: Открыть `bundle.html`, убедиться, что все React-компоненты (shadcn) отображаются корректно.

---

**Итог**: Скиллы делают вашу работу с ИИ-агентом эффективнее, позволяя переиспользовать сложные алгоритмы и стандарты оформления. Используйте их, когда задача выходит за рамки простого диалога в чате.
