Урок 10: Протокол контекста модели (MCP)

Model Context Protocol (MCP) — это фундамент интеграции данных в Antigravity. Это “USB-C порт для ИИ-приложений”, позволяющий безопасно подключать внешние данные к контексту модели.

🔌 Зачем нужен MCP?

Раньше, чтобы ИИ узнал схему вашей базы данных, приходилось копировать SQL-дампы в чат. MCP автоматизирует это. См. также Урок 11 про безопасность подключений.

Архитектура MCP:

  1. Host (Antigravity): IDE, инициирующая запросы.
  2. Client: Транспортный слой.
  3. Server: Приложение-драйвер к источнику данных (Postgres, GitHub, Google Drive).

📦 Три примитива MCP

Серверы предоставляют агенту три типа возможностей:

  1. Resources (Ресурсы): Пассивные данные для чтения.
    • Пример: Логи сервера, содержимое файлов, схемы БД.
    • URI: postgres://schema/users
  2. Tools (Инструменты): Исполняемые функции.
    • Пример: execute_sql_query, create_jira_ticket.
  3. Prompts (Промпты): Шаблоны взаимодействия.
    • Пример: “Проанализируй базу данных на ошибки”.

🛒 Магазин MCP и Конфигурация

Antigravity имеет встроенный MCP Store для быстрой установки серверов (Глава 4).

Подключение локальных серверов

Для подключения собственных утилит используется mcp_config.json. Пример подключения Docker-контейнера с Postgres:

{
  "mcpServers": {
    "local-postgres": {
      "command": "docker",
      "args": ["run", "-i", "--rm", "mcp/postgres"],
      "env": {
        "DATABASE_URL": "postgresql://user:pass@localhost/db"
      }
    }
  }
}

🌟 Context7 и Популярные Серверы

В экосистеме Antigravity выделяются несколько ключевых серверов:

1. Context7 (Deep Context)

Context7 — это мета-сервер, агрегирующий контекст из смежных сессий и внешних знаний. Он позволяет агенту “помнить” договоренности из прошлых диалогов.

Конфигурация в mcp_config.json:

"context7": {
  "command": "npx",
  "args": ["-y", "context7-server"],
  "env": {
    "API_KEY": "${env:CONTEXT7_KEY}"
  }
}

2. Топ-4 Популярных Сервера

  1. GitHub MCP: Управление репозиториями, PR и Issues прямо из чата.
  2. PostgreSQL / MySQL: Прямой SQL-доступ к данным для анализа и миграций.
  3. Google Drive: Чтение ТЗ и документации из облачных документов.
  4. Sentry: Анализ стектрейсов ошибок в реальном времени.

🔍 Верификация и Отладка

Как понять, что серверы подключены и работают?

Способ 1: Интерфейс Antigravity

  1. Откройте Settings (Настройки).
  2. Перейдите в раздел MCP Servers.
  3. Вы увидите список активных серверов с индикатором статуса (🟢 Online / 🔴 Offline).

(Примечание: используйте навык /screen для граба текущего состояния IDE)

Способ 2: Через Терминал (Shell)

Агент может проверить статус подключения через логи или прямой опрос.

Команда проверки:

# Проверка логов запуска MCP
grep "MCP Server Connected" ~/.antigravity/logs/agent.log | tail -n 5

Промпт для агента:

“Проверь, какие MCP инструменты сейчас доступны в контексте, и выведи их список.”

📝 Задание

  3. Попробуйте установить сервер GitHub (если еще не установлен) и попросить агента: “Покажи последние 5 ишью в этом репозитории”.
