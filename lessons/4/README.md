# Урок 4: Навыки Агента (Agent Skills)

Навыки — это открытый стандарт для расширения возможностей агента. Навык представляет собой папку, содержащую файл `SKILL.md` с инструкциями, которым агент следует при выполнении определенных задач.

## Что такое навыки?

Навыки — это переиспользуемые пакеты знаний, которые расширяют функционал агента. Каждый навык содержит:

- **Инструкции** о том, как подходить к конкретному типу задач.
- **Лучшие практики** и соглашения, которые следует соблюдать.
- **Дополнительные скрипты и ресурсы**, которые агент может использовать (опционально).

Когда вы начинаете диалог, агент видит список доступных навыков с их названиями и описаниями. Если навык кажется подходящим для вашей задачи, агент читает полные инструкции и следует им.

## Где хранятся навыки

Antigravity поддерживает два типа навыков:

| Расположение                                     | Область видимости                         |
| :----------------------------------------------- | :---------------------------------------- |
| `<workspace-root>/.agent/skills/<skill-folder>/` | Только для текущего рабочего пространства |
| `~/.gemini/antigravity/skills/<skill-folder>/`   | Глобальные (для всех проектов)            |

**Навыки рабочего пространства** отлично подходят для специфических рабочих процессов проекта, таких как процесс развертывания вашей команды или соглашения по тестированию.

**Глобальные навыки** работают во всех ваших проектах. Используйте их для личных утилит или инструментов общего назначения.

## Создание навыка

Чтобы создать навык:

1.  Создайте папку для вашего навыка в одной из директорий навыков.
2.  Добавьте файл `SKILL.md` внутрь этой папки.

```text
.agent/skills/
└─── my-skill/
    └─── SKILL.md
```

Каждому навыку необходим файл `SKILL.md` с YAML-заголовком (frontmatter) в верхней части:

```markdown
---
name: my-skill
description: Помогает с конкретной задачей. Используйте, когда нужно сделать X или Y.
---

# Мой навык

Здесь располагаются подробные инструкции для агента.

## Когда использовать этот навык

- Используйте это, когда...
- Это полезно для...

## Как использовать

Пошаговое руководство, соглашения и шаблоны, которым должен следовать агент.
```

### Поля заголовка (Frontmatter)

| Поле            | Обязательно | Описание                                                                                                             |
| :-------------- | :---------- | :------------------------------------------------------------------------------------------------------------------- |
| **name**        | Нет         | Уникальный идентификатор навыка (строчными буквами, дефисы вместо пробелов). По умолчанию используется имя папки.    |
| **description** | Да          | Четкое описание того, что делает навык и когда его использовать. Это то, что видит агент, решая, применить ли навык. |

> [!TIP]
> Пишите описание в третьем лице и включайте ключевые слова, которые помогут агенту распознать релевантность навыка. Например: "Генерирует юнит-тесты для кода на Python, используя соглашения pytest".

## Структура папки навыка

Хотя `SKILL.md` является единственным обязательным файлом, вы можете включить дополнительные ресурсы:

```text
.agent/skills/my-skill/
├─── SKILL.md       # Основные инструкции (обязательно)
├─── scripts/       # Вспомогательные скрипты (опционально)
├─── examples/      # Примеры реализации (опционально)
└─── resources/     # Шаблоны и другие активы (опционально)
```

Агент может читать эти файлы при выполнении инструкций вашего навыка.

## Как агент использует навыки

Навыки работают по принципу прогрессивного раскрытия:

1.  **Обнаружение**: В начале разговора агент видит список доступных навыков (имена и описания).
2.  **Активация**: Если навык кажется релевантным вашей задаче, агент считывает полное содержание `SKILL.md`.
3.  **Исполнение**: Агент следует инструкциям навыка в процессе работы над вашей задачей.

Вам не нужно явно указывать агенту использовать навык — он принимает решение на основе контекста. Тем не менее, вы можете упомянуть навык по имени, если хотите гарантировать его использование.

## Лучшие практики

- **Фокусируйтесь на одном**: Каждый навык должен делать что-то одно, но делать это хорошо. Вместо одного навыка "для всего", создайте несколько отдельных навыков для разных задач.
- **Пишите четкие описания**: Описание — это то, как агент определяет, полезен ли навык. Будьте точны в том, что он делает.
- **Используйте скрипты как "черные ящики"**: Если ваш навык включает скрипты, поощряйте агента сначала запускать их с флагом `--help`, прежде чем читать весь исходный код. Это помогает держать контекст сфокусированным на задаче.
- **Включайте деревья решений**: Для сложных навыков добавьте раздел, который поможет агенту выбрать правильный подход в зависимости от ситуации.

## Пример использования: `algorithmic-art`

Навык `skills/algorithmic-art` предназначен для создания генеративного искусства с использованием библиотеки p5.js. Он направляет агента сначала на создание "алгоритмической философии", а затем на реализацию этой философии в коде.

**Пример запроса пользователя:**

> "Создай генеративное искусство на тему 'Глубоководная биолюминесценция'"

### 1. Результат работы навыка: Алгоритмическая философия

Агент сначала создает документ с описанием концепции:

> **Название движения**: "Эмерджентное сияние"
>
> **Философия**: Мы исследуем танец света в абсолютной темноте. Красота не в самих существах, а в следах, которые они оставляют в толще воды.
>
> **Алгоритмическое выражение**: Реализация через поля потоков (flow fields), управляемые многослойным шумом Перлина. Тысячи светящихся частиц движутся по векторным траекториям, оставляя за собой затухающие шлейфы. Использование аддитивного смешивания цветов для создания эффекта свечения.

### 2. Результат работы навыка: Интерактивный арт

Затем агент генерирует самодостаточный HTML-файл с кодом p5.js.

![Пример генеративного искусства](/home/g/antigravity/lessons/4/bioluminescence_art.png)
_Визуализация алгоритма "Эмерджентное сияние", созданная с помощью навыка algorithmic-art._

Пользователь получает интерактивный инструмент, где он может менять параметры (плотность частиц, скорость течения, цветовую гамму) и генерировать новые вариации с помощью изменения `seed`.
