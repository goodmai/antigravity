Урок 3: Продвинутые техники — @ Mentions и Workflows

В этом уроке мы разберем, как эффективно ссылаться на контекст проекта и автоматизировать рутинные задачи с помощью воркфлоу.

1. @ Mentions (Упоминания файлов)

Вы можете ссылаться на файлы проекта в инструкциях для агента или в файлах конфигурации (например, .cursorrules) с помощью символа @.

Как это работает:

  • Относительные пути: @filename — интерпретируется относительно расположения текущего файла правил.
  • Абсолютные пути: @/path/to/file.md — агент сначала ищет файл в корневой системе, а если не находит, ищет относительно корня вашего рабочего пространства (workspace/path/to/file.md).

[!TIP] Используйте @ Mentions в ваших правилах, чтобы агент всегда имел под рукой описание архитектуры, стайл-гайды или спецификации API.

2. Workflows (Рабочие процессы)

Workflows — это последовательность шагов, которые направляют агента через цепочку задач (например, запуск тестов -> сборка -> деплой).

Основные характеристики:

  • Сохраняются как .md файлы.
  • Позволяют автоматизировать сложные и повторяющиеся процессы.
  • Вызываются через слеш-команду /название-воркфлоу.

Как создать Workflow:

  1. Откройте панель Customizations (через “…” в верхнем углу панели агента).
  2. Перейдите в раздел Workflows.
  3. Нажмите + Global (доступен везде) или + Workspace (только для текущего проекта).

Как запустить:

Просто введите / в чате с агентом, и вы увидите список доступных воркфлоу. Выберите нужный и нажмите Enter.

[!NOTE] Вы можете вызывать один воркфлоу из другого! Например, /workflow-1 может содержать инструкцию: “Запусти /workflow-2”.

3. Автоматическая генерация

Агент может сам генерировать воркфлоу на основе вашей истории общения! Если вы только что выполнили серию сложных действий вручную, попросите: “Создай воркфлоу на основе наших последних действий”.

4. Best Practices (Лучшие практики)

Для создания надежных и переиспользуемых воркфлоу следуйте этим правилам:

Атомарность и Модульность

Разбивайте сложные задачи на мелкие воркфлоу. Вместо одного огромного процесса создайте несколько маленьких, которые вызывают друг друга.

Пример (/run-new): Вместо того чтобы копировать логику завершения задачи в каждый новый воркфлоу, мы создали вспомогательный инструмент:

---
description: Вызов воркфлоу /new для финализации задачи
---

1. Запустить /new

Теперь любой другой воркфлоу может просто “вызвать” /run-new или прямо /new.

Использование Турбо-режима (// turbo)

Если шаг воркфлоу безопасен (например, запуск тестов или git status), добавьте аннотацию // turbo над ним. Агент выполнит его автоматически без ожидания вашего подтверждения.

Описательные названия

Называйте файлы воркфлоу так, чтобы их было легко найти через /. Например, deploy-staging.md понятнее, чем ds.md.

Практические задания

  1. Задание 1: @ Mentions. Создайте файл в корне проекта hint.md с любым текстом. Затем создайте задачу и используйте @hint.md, чтобы агент использовал информацию из этого файла.
  2. Задание 2: Создание Workflow. Откройте панель Customizations и создайте простой воркфлоу с названием hello, который просто пишет “Привет, я готов к работе!”.
  3. Задание 3: Запуск. Вызовите созданный воркфлоу командой /hello.
  4. Задание 4: Вложенность. Создайте второй воркфлоу, который вызывает первый.
