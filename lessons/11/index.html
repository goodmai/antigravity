Урок 11: Терминал, Безопасность и Режим Turbo

Antigravity предоставляет агенту доступ к терминалу, что делает его мощным инструментом, но требует строгих мер безопасности.

🛡️ Три режима работы с терминалом

  1. Off (Ручной): Агент только предлагает команды. Вы копируете и выполняете их сами. Для параноидальной безопасности.
  2. Auto (Интеллектуальный):
    • Безопасные команды (ls, cat, echo) выполняются молча.
    • Опасные (rm, docker system prune) требуют вашего подтверждения.
  3. Turbo (Автономный): Агент выполняет всё, кроме явно запрещенных команд. Для максимальной скорости.

⛔ Списки безопасности (Allow/Deny Lists)

Даже в режиме Turbo агент ограничен списками:

  • Deny List (Черный список): Команды, заблокированные всегда.
    • Пример: rm -rf /, отправка данных на неизвестные IP.
  • Allow List (Белый список): Команды, разрешенные без подтверждения.
    • Пример: npm test, git status.

🚀 Режим Turbo: Баланс риска и скорости

Режим Turbo идеален для массового рефакторинга.

  • Сценарий: “Замени во всех 500 файлах старый логгер на новый”.
  • Риск: Ошибка агента может испортить много файлов.
  • Защита: Используйте Turbo только в чистом git-дереве, чтобы всегда можно было сделать git reset.

⌨️ Inline Commands

Используйте Cmd+I (macOS) или Ctrl+I (Win/Lin) в терминале для генерации команд на естественном языке.

Пример: “Найди все файлы логов, измененные за час, и покажи ошибки 500” -> Агент сгенерирует сложный find ... | grep ... скрипт.

📝 Задание

  1. Зайдите в настройки безопасности агента.
  2. Проверьте текущий Deny List.
  3. Попробуйте в режиме Auto попросить агента создать пустой файл, а затем удалить его. Обратите внимание на запрос подтверждения при удалении.
